### 💻 開發規範
*   **核心邏輯：** 請統一寫在 `make_move.c` 中，**禁止**修改其他檔案。
*   **介面要求：** 執行時 **不得有 UI 介面**。
*   **房號命名規則：** `組別_日期_當日第幾次加入` (例如：`01_0507_1`)

### 🛠️ 編譯與執行 (嚴格遵守)
*   **編譯指令：** 
    ```bash
    gcc main.c make_move.c -o output -lws2_32
    ```
*   **執行檔名：** 統一命名為 `output`
*   **執行方式：** 
    ```bash
    ./output.exe
    ```
    > **注意：** 比賽時會以此方式直接執行，請勿加入任何額外參數。
